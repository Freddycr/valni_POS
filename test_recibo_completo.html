Explicación de las Mejoras

Validación de Tipos de Datos: Se han implementado funciones de validación para asegurar que los valores numéricos sean correctos antes de procesarlos, evitando valores NaN.

Manejo Seguro de Valores: Los valores se convierten de forma segura usando parseFloat y parseInt, con valores por defecto en caso de fallo.

Estructura Mejorada: El recibo ahora tiene una estructura clara con secciones bien definidas y separadores visuales.

Compatibilidad con Caracteres Acentuados: Se ha añadido la meta etiqueta UTF-8 para garantizar la correcta visualización de caracteres españoles.

Diseño Optimizado: El diseño se ha adaptado específicamente para impresoras de tickets de 80mm.
